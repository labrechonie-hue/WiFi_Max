# 哥哥科技安全隐私策略
# Security Policy for Mi-Stat_Max

感谢您对“小米家用路由器增强组件 (MiWiFi-Stat_Max)”项目的关注！

哥哥科技始终将代码的极简性、健壮性以及用户的本地隐私放在首位。

我们致力于打造一个如同 Home Assistant 般稳定、可 7×24 小时运行于家庭服务器（WHS）的增强服务 (Service)。

尽量对路由器的运行影响最小化，达到和官方刷新轮询间隔相当的水平；

数据拉取采取“最小化”原则，能抓小包不抓大包、有信息则不额外请求，符合 Occam's Razor 原则。

由于本项目为纯前端（浏览器扩展/暴力猴）脚本，所有微积分运算、DOM 渲染与数据流抓取均在您的浏览器本地和局域网（LAN）内完成，

**不包含任何形式的外部遥测、不主动发起任何公网请求、绝不收集任何用户隐私数据**。

## 支持概况

通常来讲，适用小米路由器的版本为该版本的重要分支之一，且作者有能力维护。但是非致命 bug 的情况下，可能维护优先级不如核心主项目ZTE-Stat_Max及其兄弟项目 HA 集成。

除此之外，该项目隶属于 Bro-Stat_Max，是其子项目。

## 漏洞报告与反馈 (Reporting a Vulnerability)

尽管本项目不涉及与第三方服务器的数据交互，

但如果您在脚本逻辑中发现了可能导致**跨站脚本攻击 (XSS)**、**路由器后台接口越权（利用由于脚本注入导致的安全降级）** 

或 **严重的内存泄漏/死循环** 等安全问题，我们非常欢迎您向我们报告！

### 报告渠道：

1. **GitHub Issues (首选)**:
   请直接在仓库提交一个带有 `[Security]` 标签的 Issue。为了保护可能存在的漏洞不被恶意利用，如果是极高危漏洞，请在 Issue 标题注明并隐藏复现代码。
2. **QQ 交流群**:
   直接在群内 @群主 提交复现步骤。
3. **哔哩哔哩弹幕网**:
   在 [该视频](https://www.bilibili.com/video/BV1PtR7B8ECC) 评论区 @UP主。

### 您可以期待的响应：

* **确认时间**：我们将在收到报告后的 **36 小时内** 评估该漏洞，并尽快确认（总计一般不超过60小时）。
* **修复流程**：一旦漏洞被验证，我们将优先在 `main` 分支进行热修复（Hotfix），
* 并同步推送到 GitHub Releases、脚本猫 (ScriptCat) 以及其他脚本分发平台的 Update URL。
* **致谢**：对于提供有效安全反馈的极客，我们会在后续版本的更新日志 (Release Notes) 中特别鸣谢！并相互推广增加友情链接等~
  
再次感谢您帮助 哥哥科技 完善这份架构设计！
